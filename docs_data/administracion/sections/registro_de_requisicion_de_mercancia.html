Requisición de Mercancía

Documento interno para retirar (o ingresar) mercancía con destino a un departamento, cargando el costo a gasto / centro correspondiente. Funciona similar a una factura, pero: no hay cliente; se identifican Departamento Solicitante y Departamento Destino; se define si el movimiento es Salida (consumo / traslado) o Entrada (retorno / ajuste). Después de ingresar líneas y observación, se Guarda y luego se Cierra; opcionalmente se imprime.

1. Datos Generales

  • Número: consecutivo (Nueva Requisición F5).
  • Elaborado: fecha del movimiento.
  • Dpto Solicita: quien requiere la mercancía.
  • Dpto Destino: área donde quedará (o a la que se devuelve).
  • Observación: motivo (uso interno, mantenimiento, exhibición, ajuste, etc.).
  • Tipo de Movimiento (Entrada / Salida): definir para afectar inventario correctamente.

2. Detalle de Productos

  • Campos de línea: Código, Descripción, Unidad, Cantidad, Costo, Total.
  • Total: suma de costos (base para contabilización de gasto interno).
  • Verifique stock disponible antes de confirmar una Salida.
  • Para Entradas (retornos) use el costo estándar o el que indique política.

3. Acciones

  • Imprimir (F4): genera formato físico / PDF.
  • Cerrar: confirma y bloquea edición (aplica movimiento a inventario y gasto).
  • Guardar (F1): guarda en estado editable.
  • Salir: abandona el formulario.
